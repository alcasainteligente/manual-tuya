Integração com Home Assistant

⭐⭐⭐ Avançado 🤖 Home Assistant ⏱️ 8-10 minutos

Liga os dispositivos Tuya ao Home Assistant através do IFTTT, criando automações mais complexas e integrações com milhares de outros serviços.

📦 Equipamentos Necessários

  • 🤖 Home Assistant instalado Raspberry Pi, servidor, etc.
  • 📱 Dispositivos Tuya Qualquer dispositivo
  • 📱 Conta IFTTT Grátis
  • 🔗 Acesso à internet Com exposição pública (opcional)

📌 O que é Home Assistant?

Home Assistant é uma plataforma open-source de automação residencial que corre localmente na tua rede. Permite controlar praticamente qualquer dispositivo, de qualquer marca, num só local, sem depender da nuvem.

Com IFTTT, podes ligar o Home Assistant a serviços que não têm integração direta.

📱 Configuração Passo-a-Passo

Passo 1: Instalar integração Tuya no Home Assistant

Antes de usar IFTTT, tenta a integração direta:

  • No Home Assistant, vai a "Configuração" → "Integrações"
  • Adiciona "Tuya" e faz login
  • Se funcionar, não precisas de IFTTT para controlo básico
📌 A integração direta é mais rápida e local. Usa IFTTT apenas para serviços externos.

Passo 2: Configurar IFTTT no Home Assistant

No Home Assistant, precisas de ativar o suporte IFTTT:

  • Edita o ficheiro configuration.yaml
  • Adiciona:
ifttt: key: YOUR_WEBHOOK_KEY

Reinicia o Home Assistant.

Passo 3: Obter Webhook URL

O URL para receber eventos do IFTTT é:

https://YOUR_HASS_URL/api/webhook/YOUR_WEBHOOK_KEY

Se não tiveres exposto o Home Assistant à internet, precisas de um serviço como ngrok ou DuckDNS.

Passo 4: Ligar IFTTT ao Home Assistant

No IFTTT:

  • Vai a "Explore" e pesquisa por "Webhooks"
  • Conecta o serviço Webhooks

Passo 5: Criar Applet para enviar evento ao HA

Cria um Applet para enviar um evento do IFTTT para o HA:

  • If This: Escolhe um trigger (ex: Weather Underground → chuva prevista)
  • Then That: Webhooks → Make a web request
  • Configura:
URL: https://YOUR_HASS_URL/api/webhook/YOUR_KEY Method: POST Content Type: application/json Body: {"action": "chuva_prevista", "intensidade": "alta"}

Passo 6: Criar automação no Home Assistant

No Home Assistant, cria uma automação para responder ao evento:

automation: - alias: "Chuva Prevista - Fechar Estores" trigger: platform: event event_type: ifttt_webhook_received event_data: action: "chuva_prevista" action: - service: cover.close_cover entity_id: cover.estores_sala - service: notify.mobile_app data: message: "Estores fechados devido à chuva prevista"

📱 Exemplos Práticos

Exemplo 1: Segurança com câmara

Se a câmara detetar movimento à noite:

  • IFTTT: Câmara Tuya → Motion detected
  • Webhook para HA com {"action": "movimento_noite"}
  • HA: Acender luzes, enviar notificação, gravar vídeo

Exemplo 2: Previsão meteorológica

Se temperatura < 5° prevista:

  • IFTTT: Weather Underground → Low temperature
  • Webhook para HA com {"action": "geada_prevista"}
  • HA: Ligar aquecimento, proteger plantas, alerta

Exemplo 3: Integração com calendário

Se calendário tem evento "Férias":

  • IFTTT: Google Calendar → Event started
  • Webhook para HA com {"action": "ferias_inicio"}
  • HA: Ativar modo ausente, simular presença

💡 Dicas e Variações

🔐 Segurança com HTTPS

Para expor o Home Assistant à internet com segurança:

  • Usa DuckDNS + Let's Encrypt para HTTPS grátis
  • Nunca exponhas sem autenticação
  • Usa chaves fortes no webhook

🔄 Múltiplas ações num evento

No IFTTT, podes enviar um JSON com várias variáveis. No HA, podes usar essas variáveis nas automações.

📊 Registo de eventos

Todos os eventos recebidos pelo HA ficam registados. Podes usá-los para estatísticas e histórico.

🔧 Alternativa sem IFTTT

Muitos serviços já têm integração direta com HA via HACS. Verifica antes de usar IFTTT.

⚠️ Problemas Comuns e Soluções

O webhook não chega ao HA

Solução:

  • Verifica se o URL está correto
  • Testa com curl ou Postman primeiro
  • Verifica logs do HA para erros

Automação não dispara

Solução:

  • Confirma o nome do evento (case sensitive)
  • Verifica se os dados correspondem ao esperado
  • Testa com um evento simples primeiro

Problemas de segurança

Solução: Se expões o HA, usa sempre HTTPS, autenticação forte e limita o acesso por IP se possível.
