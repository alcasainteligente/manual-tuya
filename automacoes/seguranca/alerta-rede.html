Alerta de Rede/Internet

⭐⭐ Intermédio 📍 Router ⏱️ 6 minutos

Notifica se a internet cair (sistema vulnerável). Pode reiniciar o router automaticamente para tentar restabelecer a ligação. Garante que o teu sistema de segurança está sempre online.

📦 Equipamentos Necessários

  • 🔌 Tomada inteligente Para ligar o router
  • 📱 IFTTT (conta gratuita) Para monitorização
  • 📡 Router Qualquer modelo
  • 📱 App SmartLife Para controlar tomada

💡 Nota: O SmartLife não tem forma nativa de saber se a internet caiu. Precisamos de usar IFTTT ou um dispositivo sempre online (ex: um sensor que pingue).

📱 Configuração Passo-a-Passo

Passo 1: Ligar o router à tomada inteligente

Liga o router à tomada inteligente Tuya.

Dá um nome: "Tomada Router".

Certifica-te que a tomada está sempre com energia (não desligues acidentalmente).

Passo 2: Criar conta no IFTTT

Acede a ifttt.com e cria uma conta gratuita.

Instala a app no telemóvel.

Passo 3: Configurar monitorização no IFTTT

No IFTTT, procura o serviço "Email" ou "SMS".

Vamos usar um truque: muitos serviços de email enviam notificações se não receberem um ping regular.

Opção mais simples: Usa um serviço como "Uptime Robot" (gratuito) que monitoriza o teu IP público e envia alerta se ficar offline.

Passo 4: Criar applet para reiniciar o router

No IFTTT, cria um applet:

IF: "Webhooks" → "Receive a web request" com o nome "router_offline"

THEN: "SmartLife" → "Turn off" a tomada do router

THEN: Delay 30 segundos

THEN: "SmartLife" → "Turn on" a tomada do router

Guarda o URL do webhook.

Passo 5: Integrar com Uptime Robot (gratuito)

Cria conta em uptimerobot.com.

Adiciona um monitor do tipo "HTTP" para o teu IP público (ou para um serviço na tua rede).

Nas definições de alerta, adiciona um webhook com o URL que obtiveste no passo 4.

Assim, quando o monitor detetar que o serviço está offline, chama o webhook que reinicia o router.

Passo 6: Criar automação de notificação (opcional)

No IFTTT, cria outro applet:

IF: Webhook "router_offline"

THEN: Notificação push: "🌐 A internet caiu! Router a reiniciar."

💡 Dicas e Variações

🔄 Variação 1: Reinício programado

Alguns routers precisam de ser reiniciados periodicamente. Cria uma automação semanal:

  • IF: "Timing" → "Every Sunday" às 4:00
  • THEN: Tomada Router = OFF, delay 30s, ON

📱 Variação 2: Alerta por SMS

Se a internet cair e o telemóvel estiver em dados móveis, recebes SMS. Configura no IFTTT para enviar SMS quando o webhook for chamado.

🏠 Variação 3: Router secundário

Se tiveres um router 4G de backup, podes ativá-lo automaticamente quando a internet principal falha.

🔋 Variação 4: UPS para o router

Para evitar que o router desligue em falhas de energia curtas, liga-o a um UPS (fonte de alimentação ininterrupta).

⚠️ Problemas Comuns e Soluções

Uptime Robot não consegue monitorizar IP local

Solução: O Uptime Robot monitoriza pela internet, não consegue ver a tua rede local. Precisas de expor um serviço (ex: um Raspberry Pi com um servidor web simples) ou usar um serviço de monitorização local.

Router não reinicia corretamente

Solução: Alguns routers demoram mais a reiniciar. Aumenta o delay para 60 segundos entre desligar e ligar.

Falsos positivos (internet considerada offline quando está online)

Solução: Ajusta os parâmetros de monitorização no Uptime Robot (intervalo maior, mais tentativas antes de alertar).

Não tenho IP público fixo

Solução: Usa um serviço de DDNS (ex: No-IP) para ter um nome de domínio que aponta para o teu IP dinâmico. Monitoriza esse domínio.
